# 멘토링 시스템 사용 시나리오 및 UI 설계

## 1. 메인 대시보드 (페이지: `/`)
- **시나리오**:
  1. 접속 시 가장 먼저 보이는 화면.
  2. "오늘의 멘토링" 섹션에서 당일 예정된 일정을 시간순으로 확인.
  3. "활성 멘티" 숫자를 확인하여 현재 부하 파악.
- **UI 구성**:
  - **Header**: 로고, 네비게이션 (대시보드 | 캘린더 | 멘티 관리)
  - **Body**:
    - [Today's Schedule]: 카드 리스트 형태. 각 카드는 시간, 멘티명, 주제, '완료하기' 버튼 포함.
    - [Stats]: 총 멘티 수, 이번 달 예정된 세션 수 등을 간단한 수치로 표시.

## 2. 멘티 관리 (페이지: `/mentees`)
- **시나리오 (멘티 추가)**:
  1. 우측 상단 "멘티 등록" 버튼 클릭.
  2. 모달 팝업. 입력 필드: 이름, 소속/직무, 멘토링 목표.
  3. "저장" 클릭 시 목록 갱신.
- **시나리오 (멘티 삭제)**:
  1. 멘티 리스트에서 삭제 아이콘 클릭.
  2. "정말 삭제하시겠습니까?" 컨펌 모달.
  3. 확인 시 삭제 처리.
- **UI 구성**:
  - 검색 바, 필터(진행중/종료).
  - 테이블 또는 카드 형태로 멘티 정보 나열. 깔끔한 흰색 카드에 그림자 효과.

## 3. 캘린더 및 일정 관리 (페이지: `/calendar`)
- **시나리오 (일정 등록)**:
  1. 달력의 날짜 클릭.
  2. 팝업에서 멘티 선택(Dropdown), 시작 시간, 종료 시간 선택.
  3. 등록.
- **시나리오 (일정 연기 - Drag & Drop)**:
  1. 달력에 있는 일정 블록을 마우스로 잡아서 다른 날짜로 이동.
  2. 자동으로 날짜 업데이트 API 호출 및 토스트 메시지 ("일정이 11월 5일로 변경되었습니다").
- **UI 구성**:
  - 화면 꽉 찬 월간 달력.
  - 주말은 옅은 배경색 처리.
  - 일정 클릭 시 상세 모달 (시간 수정, 메모 작성, 삭제 버튼).

## 4. 멘토링 일지 (기능)**
- **시나리오**:
  1. 멘토링 완료 후 해당 일정 클릭 -> "피드백/메모" 탭 선택.
  2. 주요 내용 기록 후 저장.
- **UI 구성**:
  - 일정 상세 모달 내 탭 UI (정보 | 일지).
